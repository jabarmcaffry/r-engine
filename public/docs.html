Rebur Engine Documentation

Rebur is a multiplayer-first 3D game engine built on TypeScript. Create games where every player shares the same world, with state synchronized automatically over WebSockets in real time.

💡 New here? Open the launcher, pick any world, click Edit, and follow the Quick Start below.

Core Concepts

ConceptWhat it is
EntitiesEvery object in a game world — players, platforms, bullets, cameras. They form a parent/child hierarchy.
BehaviorsTypeScript classes attached to entities. They hold game logic and run lifecycle callbacks like onTick.
ComponentsBuilt-in engine pieces: Sprite, Rigidbody, Camera, Tilemap, etc.
RootsTop-level containers: world (shared), local (per-client), server (server-only), prefabs (templates).
WorldsA project folder with a scene file and a src/ directory of behavior scripts.

Quick Start — Basic Platformer

Build your first game in Rebur in minutes. No setup needed — everything runs in the browser.

Step 1: Open the Editor

From the launcher, click Edit on any world (try Template Project or Spaceship Demo). The editor loads with a full scene graph, inspector, and script environment ready to go.

Step 2: Navigate the Editor

  • Scene Graph (left) — lists all entities in the world.
  • Properties Panel (right) — edit the selected entity's transform and synced values.
  • Behaviors Panel (right) — add or remove behaviors from the selected entity.
  • Bottom Panel — switch between Logs, Prefabs, Tilemap, and the AI Assistant.

Step 3: Test Your Game

Click ▶ Play in the top bar to enter play mode. All players who are connected will experience the game live. Click ■ Stop to exit play mode and return to edit state.

Step 4: Modify the Level

Open the Prefabs tab at the bottom. Drag Platform prefabs into the scene. Use the transform gizmo to position them. Click entities in the Scene Graph to select and edit their properties in the Inspector.

Step 5: Write Your First Script

Right-click in the Scene Graph → New Entity. Select the entity, open the Behaviors Panel, click + Add Behavior → New Script. Name it MoveRight:

import { Behavior } from "@rebur/engine";

export default class MoveRight extends Behavior {
  onInitialize(): void {
    console.log("MoveRight behavior started!");
  }

  onTick(): void {
    this.entity.transform.position.x += 2;
  }
}

Save with Ctrl+S, click ▶ Play, and watch the entity slide right.

Step 6: Add Inspector Properties

Use @syncedValue() to expose properties in the Inspector:

import { Behavior, syncedValue } from "@rebur/engine";

export default class MoveRight extends Behavior {
  // Appears as a number field in the Inspector
  @syncedValue() speed = 2;

  onTick(): void {
    this.entity.transform.position.x += this.speed;
  }
}

Step 7: Save Your Work

Click Save (top-right) or press Ctrl+S. Rebur auto-saves the scene to the project file. Use Versions for Git-style commit/push history.

Collaboration

Rebur supports Google Docs-style real-time collaboration. Multiple developers can work on the same game world simultaneously — editing the scene graph, tweaking properties, and writing scripts together.

✅ No setup required. Collaboration is automatically available on all Rebur projects. It uses the same networking infrastructure that powers multiplayer gameplay.

How it Works

  • Share your instance URL with a teammate — they open it and are immediately in the same editor session.
  • Scene changes (adding entities, modifying transforms) sync live between all editors.
  • Script files are edited in the in-browser script editor and saved to the shared project.
  • The Logs panel shows activity from all connected clients.

Best Practices

  • Communicate before making large structural changes to the scene graph.
  • Use Version Control (Versions button) to commit your work so teammates can see a clear history.
  • Assign different areas — one person handles the scene layout, another writes scripts — to avoid conflicts.

Scenes & Roots

The scene graph is split into four top-level roots. Each root determines where an entity lives and which context runs its Behaviors.

RootLives on…What to put here
worldServer + all clientsNetwork-synced gameplay objects: terrain, NPCs, pickups, projectiles.
localEach client onlyUI, the active Camera, client-side VFX, helper scripts.
serverThe server onlyGame-wide managers, match timers, scoreboards.
prefabsEditor-only (disabled at runtime)Reusable blueprints to cloneInto another root.
📐 Think of roots as layers: world is the shared game world · local is each player's personal overlay · server is the omniscient referee · prefabs are templates you clone wherever needed.

Dot-Path Lookup (root._.<Name>)

Every root exposes a _ proxy that lets you grab children by name with dot-syntax:

// Get the Player prefab
const player = this.game.prefabs._.Player;

// Get a child named "Health" under Player
const health = this.game.prefabs._.Player._.Health;

// Get a runtime instance in world
const enemy = this.game.world._.Enemy42;

// Get the client-side camera in local
const cam = this.game.local._.Camera;
💡 Name collisions? Use bracket notation: this.game.world._["My Entity"]

Entities

Entities are the fundamental building blocks of a Rebur game. They form a parent/child hierarchy and can have Behaviors and Components attached to them.

Spawning Entities at Runtime

import { Behavior, Mesh } from "@rebur/engine";

export default class Spawner extends Behavior {
  onInitialize(): void {
    if (!this.game.isServer()) return; // only spawn on server

    const { x, y, z } = this.entity.transform.position;
    const bullet = this.game.world.spawn({
      type: Mesh,
      name: "Bullet",
      transform: {
        position: { x, y, z },
        scale: { x: 0.2, y: 0.2, z: 0.2 },
      },
      behaviors: [{ type: BulletBehavior }],
    });
  }
}

Destroying Entities

// Destroy this entity
this.entity.destroy();

// Destroy another entity
someOtherEntity.destroy();

Finding Entities

// By dot-path from root (fastest)
const player = this.game.world._.Player;

// Child of this entity by name
const gun = this.entity._.GunEntity;

// All entities with a specific behavior type
const enemies = this.game.entities.lookupByBehavior(EnemyBehavior);
⚠️ Don't use children.find() — it's inefficient. Always use dot-path syntax (entity._.ChildName) for lookups.

Entity Transform

import { Quat } from "@rebur/engine";
const t = this.entity.transform;

// Position (Vec3, world space, in metres)
t.position.x = 5;
t.position.y = 2;
t.position.z = -3;

// Rotation (Quat — use fromEulerXYZ for Euler angles in radians)
t.rotation = Quat.fromEulerXYZ(0, Math.PI / 2, 0); // 90° yaw

// Scale (Vec3)
t.scale.x = 2;
t.scale.y = 2;
t.scale.z = 2;

// Global vs local
t.position        // local-space position (Vec3)
t.globalTransform // fully computed world transform

Cloning from Prefabs

// Clone the PlayerPrefab into the world root
const player = this.game.prefabs._.PlayerPrefab.cloneInto(this.game.world);

Behaviors

Behaviors are TypeScript classes that bring logic to an Entity. If you know Unity, think MonoBehaviours; in Godot, they're Scripts.

Basic Structure

import { Behavior } from "@rebur/engine";

export default class MyBehavior extends Behavior {
  onInitialize(): void {
    // Runs once when the entity is first set up
    console.log("Initialized:", this.entity.name);
  }

  onTick(): void {
    // Runs every frame (~60 fps)
  }

  destroy(): void {
    // Clean up when entity is removed
    super.destroy();
  }
}

Lifecycle Callbacks

MethodWhen it runsUse for
onInitialize()Once, on entity setupGet references, subscribe to events, spawn children.
onInitializeClient()Client only, on setupSet up UI, play VFX, register DOM listeners.
onInitializeServer()Server only, on setupSpawn match controller, load map data.
onTick()Every frameMovement, AI, input checks.
onTickClient()Client every frameClient-only rendering, interpolation.
onTickServer()Server every frameAuthoritative physics, match logic.

Getting Other Behaviors

// Get a behavior from this entity
import { Rigidbody, Vec3 } from "@rebur/engine";
const rb = this.entity.getBehavior(Rigidbody);
if (rb) rb.linearVelocity = new Vec3(5, 0, 0);

// Cast an entity to access its component type
import { Mesh } from "@rebur/engine";
const mesh = this.entity._.MeshChild.cast(Mesh);
mesh.color = "#ff0000";

Platform-Specific Hooks

Server vs Client — Make dependencies explicit; never rely on tick ordering between behaviors. Use this.game.isServer() / this.game.isClient() guards when behavior differs by context. 
onInitialize(): void {
  if (this.game.isServer()) {
    // Server-only setup
  }
  if (this.game.isClient()) {
    // Client-only setup
  }
}

Tick Loop

The tick loop is the heartbeat of Rebur. onTick() fires every frame at the server's tick rate. Client frames are interpolated for smooth visuals.

Delta Time

Use this.time.delta (seconds since last tick) for frame-rate-independent movement:

onTick(): void {
  const dt = this.time.delta; // seconds
  const speed = 5; // metres per second
  this.entity.transform.position.x += speed * dt;
}

Timer Pattern

import { Behavior, syncedValue } from "@rebur/engine";

export default class PingTimer extends Behavior {
  private elapsed = 0;
  @syncedValue() interval = 2; // seconds

  onTick(): void {
    this.elapsed += this.time.delta;
    if (this.elapsed >= this.interval) {
      this.elapsed = 0;
      console.log("Ping! (every", this.interval, "seconds)");
    }
  }
}

Time API

PropertyTypeDescription
this.time.deltanumberSeconds since last tick.
this.time.nownumberTotal elapsed time in seconds since world start.
this.game.ticknumberCurrent tick count (integer).

Synced Values & Adapters

@syncedValue() turns a field into: a network-replicated variable, an observable you can onChanged(), and a row in the Inspector.

Basic Usage

import { Behavior, syncedValue } from "@rebur/engine";

export default class MyBehavior extends Behavior {
  @syncedValue() myNumber = 10;
  @syncedValue() myBoolean = false;
  @syncedValue() myString = "hello";
}

Changes made in the Inspector apply to the instance, and synced values are replicated over the network automatically.

Listening for Changes

onInitialize(): void {
  this.values.get("myNumber")?.onChanged((next, prev) => {
    console.log(`Changed from ${prev} to ${next}`);
  });
}

  @syncedValue(shapeAdapter) shape: Shape = "circle";
}

Logging

console.log() and its variants are automatically captured on both the server and the client, and streamed into the in-game Logs panel — no setup required.

Basic Usage

if (this.game.isServer()) {
  console.log("Hello from the server!");
}

if (this.game.isClient()) {
  console.log("Hello from client!");
}

Log Capture Details

  • On the server, all output from STDOUT and STDERR is captured and forwarded to connected clients.
  • On the client, only console.log() is captured to the Logs panel.
  • Use console.info(), console.warn(), or console.error() on the client to bypass capture and write to the browser's DevTools console instead.
💡 Debug tip — Open the browser DevTools console (F12) alongside the Logs panel for full output visibility.

Camera

The Camera entity is a 3D perspective camera. Place it in the local root so each player has an independent view. In transform mode (default for games), the camera renders from its own entity transform — parent it to a character rig, or move and rotate it in a Behavior to control the view.

Third-Person Follow Camera

import { Behavior, syncedValue } from "@rebur/engine";

export default class CameraFollow extends Behavior {
  @syncedValue() smoothing = 0.1;
  @syncedValue() heightOffset = 5;
  @syncedValue() depthOffset = 10;

  onTick(): void {
    const player = this.game.world._.Player;
    if (!player) return;

    const target = player.transform.position;
    const cam = this.entity.transform.position;
    const s = this.smoothing;

    // Smooth follow — interpolate each axis independently
    cam.x += (target.x              - cam.x) * s;
    cam.y += (target.y + this.heightOffset - cam.y) * s;
    cam.z += (target.z + this.depthOffset  - cam.z) * s;
  }
}
💡 Instant attach — For a simple follow cam with no scripting, parent the Camera entity directly to the player entity in the scene graph. It will move with the player automatically.

Switching Between Cameras

import { Behavior, Camera } from "@rebur/engine";

export default class CameraSwitch extends Behavior {
  onInitialize(): void {
    const camA = this.game.local._.CameraA.cast(Camera);
    const camB = this.game.local._.CameraB.cast(Camera);

    camA.activate(); // sets active = true and registers with renderer
    camB.active = false;
  }
}

Camera Properties

PropertyDescription
camera.activeWhether this camera drives the viewport.
camera.fovVertical field of view in degrees (default 75). Lower = zoomed in, higher = wide-angle.
camera.nearNear clipping plane distance (default 0.1 m).
camera.farFar clipping plane distance (default 1000 m).
camera.orbitOrbit mode — camera orbits a focus point at a zoom distance. Used by the editor viewport; set to false for game cameras.
camera.activate()Set this camera as the active viewport camera.
📍 Camera lives in local — Each client controls its own camera. Place your Camera entity under the local root so each player has an independent view. For first-person, parent the Camera to the player at eye height. For third-person, use a follow behavior with a position offset.

Remote Procedure Calls (RPC)

RPCs let behaviors on the server call methods on the client (and vice versa) across the network. They're how you send targeted messages — like "play this sound for this player" or "trigger this animation."

Defining an RPC

import { Behavior, rpc } from "@rebur/engine";

export default class PlayerBehavior extends Behavior {
  // Called on client by the server — plays a hit effect
  @rpc.server_to_client()
  onHit(damage: number): void {
    console.log(`Took ${damage} damage!`);
    // play VFX, shake camera, etc.
  }

  // Called on server by a client — request to fire a bullet
  @rpc.client_to_server()
  requestFire(direction: { x: number; y: number }): void {
    // server validates and spawns the bullet
    this.spawnBullet(direction);
  }
}

Calling RPCs

// Server calls a method on ALL clients:
this.onHit.broadcast(25);

// Server calls a method on ONE specific client:
this.onHit.call(connectionId, 25);

// Client calls a method on the server:
this.requestFire.call("server", { x: 1, y: 0 });

RPC Types

DecoratorDirectionUse for
@rpc.server_to_client()Server → Client(s)Trigger client-side effects, UI updates.
@rpc.client_to_server()Client → ServerRequest an action, validate player input.

Tilemaps

Tilemaps let you paint levels quickly using a tileset (a grid of sprites) or colored-square tiles. A Tilemap entity is a regular entity — move, scale, parent, and duplicate it like anything else.

Quick Start

  1. Right-click in the Scene Graph → New Entity → Tilemap.
  2. In the Inspector, drag a spritesheet PNG onto Atlas. The editor auto-slices it into tiles (64×64 px by default).
  3. Select the Tilemap entity, open the Tilemap tab at the bottom.
  4. Pick a tile and start painting!
🎨 Scale filter — Set "Scale filter mode" to nearest for crisp pixel art, or linear for smooth scaling.

Scripting a Tilemap

import { Behavior, Tilemap } from "@rebur/engine";

export default class ProceduralLevel extends Behavior {
  onInitialize(): void {
    const map = this.entity.cast(Tilemap);

    // Set a tile (x, y, paletteId)
    map.setTile(0, 0, 1);
    map.setTile(1, 0, 1);
    map.setTile(2, 0, 2);

    // Erase a tile
    map.setTile(3, 0, undefined);

    // Read a tile
    const tile = map.getTile(0, 0);

    // Iterate all filled cells
    for (const { x, y, tile } of map.tiles()) {
      console.log(x, y, tile);
    }

    // Clear everything
    map.clearTiles();
  }
}

Tilemap Methods

MethodDescription
setTile(x, y, paletteId?)Draw or erase a cell. undefined = erase.
getTile(x, y)Return tile data or undefined.
getTileAtPoint(worldPos)Find tile at a world-space position.
clearTiles()Erase all tiles.
tiles()Generator — iterate { x, y, tile } of filled cells.

Audio & Sounds

Play sounds using the Sound component. Audio works in the browser — upload .ogg, .mp3, or .wav files to your project assets.

Playing a Sound

import { Behavior, Sound } from "@rebur/engine";

export default class SoundPlayer extends Behavior {
  onInitialize(): void {
    if (!this.game.isClient()) return;

    const sound = this.entity._.SoundEntity.cast(Sound);
    sound.src = "res://assets/explosion.ogg";
    sound.play();
  }
}

Using @syncedValue with AudioAdapter

import { Behavior, syncedValue, AudioAdapter, Sound } from "@rebur/engine";

export default class AudioBehavior extends Behavior {
  @syncedValue(AudioAdapter) audioFile = "res://assets/music.ogg";
  @syncedValue() loop = true;
  @syncedValue() volume = 1.0;

  onInitialize(): void {
    if (!this.game.isClient()) return;
    const sound = this.entity.cast(Sound);
    sound.src = this.audioFile;
    sound.loop = this.loop;
    sound.volume = this.volume;
    sound.play();
  }
}
🔊 Client-only — Sound playback is always client-side only. Guard audio code with this.game.isClient().

Key-Value (KV) Storage

The KV store is a persistent, server-side key-value database. Use it for player save data, leaderboards, global game state, and anything that needs to survive restarts.

KV Scopes

ScopeWho can accessUse for
this.game.kv.serverServer onlyGlobal game state, leaderboards, admin data.
this.game.kv.playerPlayers (own scope); Server (any player)Per-player scores, inventory, preferences.

Basic KV Operations

import { Behavior } from "@rebur/engine";

export default class SaveData extends Behavior {
  async onInitialize(): Promise<void> {
    if (!this.game.isServer()) return;

    // Server scope
    await this.game.kv.server.set("totalClicks", 0);
    const clicks = await this.game.kv.server.get("totalClicks");

    // Player scope
    const playerId = "player_123";
    await this.game.kv.player.set("score", 100, playerId);
    const score = await this.game.kv.player.get("score", playerId);

    // List all player IDs with stored data
    const allPlayers = await this.game.kv.info.players();
  }
}

Available Methods

MethodDescription
kv.server.get(key)Get a global value.
kv.server.set(key, value)Set a global value.
kv.server.list()List all global keys.
kv.server.delete(key)Delete a global key.
kv.player.get(key, playerId)Get a player's value.
kv.player.set(key, value, playerId)Set a player's value.
kv.player.list(playerId)List all keys for a player.
kv.player.delete(key, playerId)Delete a player's key.
kv.player.clear(playerId)Clear all keys for a player.
kv.info.players()List all player IDs with stored data.
🔍 KV Debugger — Attach the KVDebugger behavior to a UILayer entity in your game to get a visual interface for inspecting and modifying stored data during development.

Network Authority

In Rebur, the server is the authority. Entities spawned on the server replicate automatically to all clients. Clients can request actions via RPCs, but the server validates and executes them.

Authority Model

ContextCan spawn entities?Data is synced?
Server✅ Yes — synced to all clients✅ Authoritative source of truth
Client✅ But only in local rootLocal only — not sent to server or others

Checking Context in Code

onInitialize(): void {
  if (this.game.isServer()) {
    // This runs only on the server
    // Spawned entities here will be synced to clients
  }

  if (this.game.isClient()) {
    // This runs only on clients
    // Good for UI, VFX, local state
  }
}

Identifying Connected Players

// Get all connected players
const connections = this.game.network.connections;

for (const conn of connections) {
  console.log(conn.id, conn.playerId, conn.nickname);
}

// Get the local player's connection ID
const myId = this.game.network.self;

Sending Custom Messages

// Client → Server (custom message)
this.game.network.sendCustomMessage("server", "@game/action", { type: "jump" });

// Server → All clients
this.game.network.sendCustomMessage("broadcast", "@game/event", { event: "levelUp" });

Physics

Rebur uses the Rapier3D physics engine for fast, deterministic simulation. Add physics to entities using Collider and Rigidbody entity types.

Colliders

Collider entities define the physical shape. They are static by default — great for walls, floors, and obstacles.

  • Right-click in Scene Graph → New Entity → Collider
  • Choose shape (box, sphere, capsule) in the Inspector
  • Adjust size, offset, and physics material

Rigidbodies

A Rigidbody entity needs at least one Collider child to define its shape. Rigidbodies are simulated dynamically.

import { Behavior, Rigidbody, Vec3 } from "@rebur/engine";

export default class PhysicsPlayer extends Behavior {
  private rb!: Rigidbody;

  onInitialize(): void {
    this.rb = this.entity.cast(Rigidbody);
  }

  onTick(): void {
    const input = this.inputs;
    const vel = this.rb.linearVelocity;
    if (input.getKey("KeyA")) this.rb.linearVelocity = new Vec3(-5, vel.y, vel.z);
    if (input.getKey("KeyD")) this.rb.linearVelocity = new Vec3( 5, vel.y, vel.z);
  }
}

Character Controller

Character Controller is a special kinematic entity. It doesn't simulate gravity on its own — you control its movement, and the physics engine handles collision response.

import { Behavior, CharacterController } from "@rebur/engine";

export default class PlayerController extends Behavior {
  private cc!: CharacterController;

  onInitialize(): void {
    this.cc = this.entity.cast(CharacterController);
  }

  onTick(): void {
    const dt = this.time.delta;
    this.cc.move(new Vec3(3 * dt, 0, 0)); // move right, physics handles collisions
  }
}

Collision Events

import { Behavior, EntityCollision } from "@rebur/engine";

export default class CollisionHandler extends Behavior {
  onInitialize(): void {
    this.listen(this.entity, EntityCollision, ({ started, other, contactPoint, normal }) => {
      if (started) {
        console.log("Collided with:", other.name);
      }
    });
  }
}

Ray Casting

import RAPIER from "@rebur/vendor/rapier.ts";

// Cast a ray upward from this entity's position
const pos = this.entity.transform.position;
const ray = new RAPIER.Ray(
  { x: pos.x, y: pos.y, z: pos.z },   // origin
  { x: 0, y: 1, z: 0 }               // direction (upward)
);
const hit = this.game.physics.world.castRay(ray, 4.0, true);
if (hit) {
  const entity = this.game.entities.lookupByRef(hit.collider.userData.entityRef);
  console.log("Hit:", entity?.name, "at distance", hit.timeOfImpact);
}

Preloading Assets

Preload textures, sounds, and other assets before the game starts to avoid loading hitches during gameplay.

preload.ts

Create a preload.ts file in the root of your project:

export default definePreload({
  textures: [
    "res://assets/player.png",
    "res://assets/tileset.png",
    "res://assets/explosion.png",
  ],
  sounds: [
    "res://assets/music.ogg",
    "res://assets/hit.ogg",
  ],
  custom: async () => {
    // any custom async preloading
  },
});

Importing npm Libraries

Import external npm packages using the npm: prefix:

import nipplejs from "npm:nipplejs";
import gsap from "npm:gsap";

The package is automatically fetched and loaded when your script runs. This works with any npm package.

Asset Path Format

Use the res:// prefix for assets in your project's asset folder:

  • res://assets/player.png — image in the assets/ folder
  • res://sounds/explosion.ogg — audio file
  • res://data/config.json — any other file

To resolve a resource path at runtime:

const url = this.game.resolveResource("res://assets/player.png");

User Interface

Build HUDs, menus, and overlays using UIBehavior — a JSX-based system that renders into an isolated Shadow DOM layer on top of the game canvas.

UILayer vs UIPanel

ComponentUse for
UILayerFull-screen overlay (HUDs, menus, inventory). Fixed to the screen.
UIPanelAnchored to a world entity (nameplates, speech bubbles, shop signs). Follows the entity in world space.

Basic UIBehavior

import { UIBehavior } from "@rebur/engine";

export default class ScoreDisplay extends UIBehavior {
  score = 0;

  render() {
    return (

PlatformMovement Behavior

import { Behavior, syncedValue, CharacterController, Vec3 } from "@rebur/engine";

export default class PlatformMovement extends Behavior {
  @syncedValue() speed = 5;
  @syncedValue() jumpForce = 8;
  @syncedValue() gravity = 20;

  private cc!: CharacterController;
  private velocityY = 0;

  onInitialize(): void {
    this.cc = this.entity.cast(CharacterController);
  }

  onTick(): void {
    if (!this.game.isServer()) return;
    const dt = this.time.delta;
    const input = this.inputs;

    let vx = 0, vz = 0;
    if (input.getKey("KeyA") || input.getKey("ArrowLeft"))  vx = -this.speed;
    if (input.getKey("KeyD") || input.getKey("ArrowRight")) vx =  this.speed;
    if (input.getKey("KeyW") || input.getKey("ArrowUp"))   vz = -this.speed;
    if (input.getKey("KeyS") || input.getKey("ArrowDown"))  vz =  this.speed;

    this.velocityY += this.gravity * dt;
    if (this.cc.isOnGround && input.getKeyDown("Space")) {
      this.velocityY = -this.jumpForce;
    }

    this.cc.move(new Vec3(vx * dt, -this.velocityY * dt, vz * dt));
  }
}

Ingredient: Basic Structure

The complete anatomy of a Behavior class — what every script you write in Rebur should look like.

import { Behavior, syncedValue } from "@rebur/engine";

export default class MyBehavior extends Behavior {
  // 1. Synced values appear in the Inspector and replicate over the network
  @syncedValue() speed = 5;
  @syncedValue() label = "Hello";
  @syncedValue() active = true;

  // 2. Private state — not synced, stays local to this behavior instance

    console.log("Initialized on entity:", this.entity.name);
    // Listen for events here
  }

  onTick(): void {
    this.timer += this.time.delta;
  }

  // 5. destroy — clean up when entity is removed
  destroy(): void {
    // cancel timers, remove DOM elements, etc.
    super.destroy();
  }
}

Ingredient: Character Controller

A complete movement controller using CharacterController with WASD/arrow key input, normalization for diagonal movement, and delta-time scaling.

import { Behavior, syncedValue, CharacterController, Vec3 } from "@rebur/engine";

export default class CharacterMovement extends Behavior {
  @syncedValue() speed = 5;
  private cc!: CharacterController;

  onInitialize(): void {
    this.cc = this.entity.cast(CharacterController);
  }

  onTick(): void {
    if (!this.game.isServer()) return;
    const input = this.inputs;
    let vx = 0, vz = 0;

    if (input.getKey("KeyA") || input.getKey("ArrowLeft"))  vx -= 1;
    if (input.getKey("KeyD") || input.getKey("ArrowRight")) vx += 1;
    if (input.getKey("KeyW") || input.getKey("ArrowUp"))   vz -= 1;
    if (input.getKey("KeyS") || input.getKey("ArrowDown"))  vz += 1;

    // Normalize so diagonals aren't faster
    const dir = new Vec3(vx, 0, vz);
    const len = dir.length();

  }
}

Ingredient: Interacting with Behaviors

Access and call methods on behaviors attached to other entities using getBehavior().

import { Behavior, EntityCollision } from "@rebur/engine";
import HealthBar from "./HealthBar.ts";

import RAPIER from "@rebur/vendor/rapier.ts";
import { Behavior, CharacterController } from "@rebur/engine";

export default class RaycastExample extends Behavior {

    const origin = this.entity.transform.position;
    const direction = { x: 0, y: -1, z: 0 }; // cast downward (negative Y in 3D)

    const ray = new RAPIER.Ray(origin, direction);
    const maxDistance = 4.0;

    const hit = this.game.physics.world.castRay(

      undefined, undefined, selfCollider

    if (hit) {
      const entity = this.game.entities.lookupByRef(hit.collider.userData.entityRef);
      console.log("Hit:", entity?.name, "at distance", hit.timeOfImpact);
    }
  }
}

Ingredient: Quaternion (Quat) API

Quaternions represent 3D rotations without gimbal lock. Use Quat for any entity rotation in Rebur.

import { Quat } from "@rebur/engine";

// From Euler angles (radians: X pitch, Y yaw, Z roll)
const q = Quat.fromEulerXYZ(Math.PI / 4, 0, 0); // 45° pitch

// From axis + angle
const yaw = Quat.fromAxisAngle({ x: 0, y: 1, z: 0 }, Math.PI / 2);

// Compose rotations (apply r2 after r1)
const combined = r1.multiply(r2);

// Smooth rotation (slerp)
const mid = q1.slerp(q2, 0.1); // 10% of the way from q1 to q2

// Rotate a direction vector
const forward = q.rotateVec3({ x: 0, y: 0, z: -1 }); // camera forward

// Spin entity around Y-axis each tick
onTick(): void {
  const spin = Quat.fromAxisAngle({ x: 0, y: 1, z: 0 }, this.time.delta * 2);
  this.entity.transform.rotation = spin.multiply(this.entity.transform.rotation);
}

API Reference

Method / FactoryDescription
Quat.fromEulerXYZ(x, y, z)Create from Euler angles in radians (pitch, yaw, roll).
